## Log Files

### test_cycle_YYYYMMDD_HHMMSS.log
Complete test execution log with timestamps, including:
- Test discovery phase
- Execution progress
- Results summary
- Coverage analysis
- Performance metrics

### test_inventory.json
JSON file containing discovered tests before execution:

### coverage_report.html
HTML coverage report generated by pytest-cov, showing:
- Line-by-line coverage
- Branch coverage
- Missing lines
- Coverage percentages per file

## Stage Summaries

Detailed logs for each test category:

- **unit_tests.log**: Unit test execution details
- **integration_tests.log**: Integration test results
- **architecture_tests.log**: Architecture compliance validation
- **health_tests.log**: Health monitoring test results
- **performance_tests.log**: Performance benchmark results

## Log Retention

- Logs are rotated automatically
- Keep last 30 days of test cycles
- Coverage reports kept for last 10 runs
- Stage summaries archived monthly

## Usage

Logs are automatically generated when running:
```bash
python test.py --coverage
```

To view recent test results:
```bash
# View latest test cycle
tail -f test_logs/test_progress.log

# Check coverage
open test_logs/coverage_report.html

# Review specific stage
cat test_logs/stage_summaries/unit_tests.log
